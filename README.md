# 2026-VoidLog
우주 퍼즐 인디 게임 &lt;VoidLog> 레포지토리

## 폴더 구조
직접 만드는 모든 작업물은 `_Project/` 폴더 내부에서만 관리합니다.
```text
Assets/
├── _Project/
│   ├── Animations/   # 애니메이션 컨트롤러 및 애니메이션 클립
│   ├── Art/          # 스프라이트, 일러스트, UI 소스 등 원본 이미지
│   ├── Audio/        # 배경음(BGM), 효과음(SFX) 등 사운드 소스
│   ├── Prefabs/      # 재사용 가능한 프리팹 (Scripts 구조와 매칭)
│   ├── Scenes/       # 메인 게임, 엔딩 컷씬 등 주요 씬
│   └── Scripts/      # 기능 명세서 기반 스크립트 폴더
│       ├── Core/     # 게임 루프, 데이터 매니저 등 시스템 코드
│       ├── Office/   # (DSK_) 노이즈 퍼즐, 보류함 등 '업무' 기능
│       ├── Player/   # 플레이어 조작, 이동, 입력 포커스 처리
│       ├── Puzzle/   # (ESC_) 암호/퍼즐 기능
│       ├── System/   # (SYS_) 전력 소모, 하루 카운트, 정산 시스템
│       └── UI/       # (UI_) HUD 연출, 엔딩 분기 스크린 전환
└── ThirdParty/       # 에셋 스토어 등 외부 플러그인 (터치 금지!)
```

## Commit 규칙<br>
- **Feat** | 코드 파일이나 부수적인 코드 추가 |<br>
- **Fix** | 코드 및 내부 파일 수정 |<br>
- **Style** | 문법 오류 해결, 이름 변경 등 코드 자체의 변경이 없는 수정 |<br>
- **Del** | 쓸모없는 코드 혹은 파일 삭제 |<br>
- **Move** | 프로젝트 파일 및 코드 위치 변경 |<br>
- **Rename** | 파일 이름 변경 |<br>
- **Merge** | 브랜치 병합 및 충돌 해결 |<br>
- **Refactor** | 코드 전면 수정 |<br>

## 협업 방식 (GitHub Flow)
1. **브랜치 규칙**: `feature/기능명` 또는 `fix/기능명` 사용
2. **작업 순서**: 
   - `main`에서 최신 코드 `pull`
   - 새로운 브랜치 생성 후 작업
   - 작업 완료 후 `push` 및 **Pull Request(PR)** 생성
   - 팀원 리뷰 후 `merge`
  
## 필독 사항
1. **작업물 위치**: 본인 작업물은 반드시 `_Project/` 내부의 알맞은 하위 폴더에 생성합니다.
   - 외부 플러그인이나 무료 에셋을 가져올 때만 `ThirdParty/` 폴더를 이용합니다.

2. **빈 폴더 유지**: 빈 폴더가 Git에서 유실되지 않도록 `.gitkeep` 파일은 함부로 삭제하지 마세요.
