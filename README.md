# 2026-VoidLog
우주 퍼즐 인디 게임 &lt;VoidLog> 레포지토리

## Commit 규칙<br>
| **Feat** | 코드 파일이나 부수적인 코드 추가 |<br>
| **Fix** | 코드 및 내부 파일 수정 |<br>
| **Style** | 문법 오류 해결, 이름 변경 등 코드 자체의 변경이 없는 수정 |<br>
| **Del** | 쓸모없는 코드 혹은 파일 삭제 |<br>
| **Move** | 프로젝트 파일 및 코드 위치 변경 |<br>
| **Rename** | 파일 이름 변경 |<br>
| **Merge** | 브랜치 병합 및 충돌 해결 |<br>
| **Refactor** | 코드 전면 수정 |<br>

## 협업 방식 (GitHub Flow)
1. **브랜치 규칙**: `feature/기능명` 또는 `fix/기능명` 사용
2. **작업 순서**: 
   - `main`에서 최신 코드 `pull`
   - 새로운 브랜치 생성 후 작업
   - 작업 완료 후 `push` 및 **Pull Request(PR)** 생성
   - 팀원 리뷰 후 `merge`
